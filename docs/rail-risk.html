Project Overview

This project predicts the likelihood of highway-rail crossing accidents across the United States under varying weather conditions by integrating historical accident data from the Federal Railroad Administration (FRA) with granular hourly weather data from the NOAA National Centers for Environmental Information (NCEI) API.

The pipeline assigns up to 5 nearest weather stations to each accident location, downloads hourly weather data spanning 50 years (1975–2025), and performs timezone-aware merging to match accidents with weather conditions at the time and place they occurred.

Why This Project Matters

  • Highway-rail grade crossing accidents result in hundreds of deaths and injuries annually in the US.
  • Understanding the role of weather (visibility, precipitation, temperature, wind) can inform safety policy.
  • Enables predictive models for high-risk weather conditions.
  • Supports resource allocation for railroad safety programs.
  • Drives public awareness campaigns about weather-related crossing dangers.

Pipeline — 4 Phases

Phase 1 — Station Assignment (~5 min)

Assigns up to 5 nearest WBAN weather stations to each accident using KDTree spatial indexing with a 50-mile max distance. All 228,847 accidents had at least 1 station nearby.

Phase 2 — Prepare Weather Requests (~2 min)

Creates a deduplicated list of unique (station, date) pairs. Reduces ~1.1M potential requests down to 625,820 unique pairs — a 44% reduction.

Phase 3 — Download Weather Data (6–8 hrs)

Bulk downloads hourly weather CSVs from NOAA Global Hourly dataset. Downloads ~10,000 station-year files, extracting 1.3M+ hourly observations total.

Phase 4 — Merge Accidents with Weather (~15 min)

Timezone-aware temporal matching. Converts accident local time → UTC, then finds nearest weather observation within ±60 minutes using multi-station fallback.

Key Challenges & Solutions

🔌 NOAA API Reliability

Initial approach using NOAA CDO v2 API returned HTTP 500 errors. Pivoted to bulk CSV downloads from NOAA Global Hourly — far more reliable for 50-year historical data at scale.

📍 Station Data Availability

Single station assignment gave only 13% match rate due to historical gaps. Using 5 stations per accident with fallback logic raised this to 40–70%.

🕐 Timezone Complexity

Accidents recorded in local time, weather in UTC. Used timezonefinder + pytz for accurate DST-aware conversion before temporal matching.

⚡ Scale & Performance

228k accidents × 75k stations = 17 billion naive comparisons. KDTree spatial indexing reduced this to O(n log m), completing station assignment in ~5 minutes instead of 10+ hours.
