Project Overview

Housing prices are a key indicator affecting investment decisions and market trends. This project analyzes county-level housing price trends using Zillow's Housing Value Index (ZHVI) dataset and develops a predictive model to forecast future housing prices using Apache Spark and PySpark for scalable ML pipelines.

The study focuses on 5 states (NJ, NY, PA, CA, TX) and investigates how well recent price trends — captured through lag features — can predict short-term future housing prices.

Dataset

The dataset covers middle-tier housing prices (33rd–67th percentile) for single-family homes and condos, with seasonally adjusted and smoothed values from Zillow. Each row originally represented a county with monthly price columns from January 2000 to December 2025.

Geographic Scope

5 states: New Jersey, New York, Pennsylvania, California, Texas

Time Range

2010–2025 (focused on modern market trends, avoiding early missing values)

Format

Wide format (one column per month) → transformed to long format (one row per county-month)

Final Size

78,633 county-month observations after preprocessing and missing value removal

Spark ML Pipeline

1

State & Time Filtering

Retained only counties from 5 selected states. Selected monthly price columns from 2010 onwards to reduce missing values and focus on modern market trends.

2

Wide-to-Long Transformation

Used Spark's stack() function to reshape data from wide format (one column per month) into long time-series format with fields: RegionID, RegionName, State, Metro, Date, ZHVI price.

250,090 raw → 78,633 clean county-month observations
3

Feature Engineering — Lag Features

Used Spark window functions partitioned by RegionID and ordered by date to create time-based features independently per county. Target variable: next month's ZHVI via lead().

lag_1 · lag_3 · lag_12 features created
4

Time-Based Train/Test Split

Preserved temporal structure — no random shuffling. Training: 2010–2022 (57,009 rows, 78.34%). Testing: 2023–2025 (15,761 rows, 21.66%).

5

VectorAssembler + Model Training

Combined lag_1, lag_3, lag_12 into a single feature vector using VectorAssembler. Trained 3 Spark MLlib models and evaluated with RMSE and MAE.

Linear Regression · Gradient Boosted Trees · Random Forest

Lag Features Explained

lag_1

Price from 1 month prior — captures very recent price momentum

lag_3

Price from 3 months prior — captures short-term trends

lag_12

Price from 12 months prior — captures seasonal/yearly patterns (same month last year)

Target (label)

Next month's ZHVI price, created with Spark's lead() window function

# Spark window function for lag features
from pyspark.sql import Window
from pyspark.sql.functions import lag, lead

window = Window.partitionBy("RegionID").orderBy("Date")

df = df.withColumn("lag_1", lag("ZHVI", 1).over(window))
df = df.withColumn("lag_3", lag("ZHVI", 3).over(window))
df = df.withColumn("lag_12", lag("ZHVI", 12).over(window))
df = df.withColumn("label", lead("ZHVI", 1).over(window))

# VectorAssembler for feature vector
assembler = VectorAssembler(inputCols=["lag_1", "lag_3", "lag_12"], outputCol="features")

Why Linear Regression won: Housing prices usually change gradually over time, creating a simple and consistent relationship between past and future prices. Tree-based models rely on decision splits — they're better for complex nonlinear relationships but struggle to follow smooth time trends like housing prices.

Conclusion & Future Work

The project demonstrated that recent housing prices are strong predictors of short-term price movements. The lag features captured recent trends and seasonal patterns effectively, helping Linear Regression achieve low prediction error. The Spark ML pipeline scaled efficiently across 78,633 county-month observations.

Future work could include additional variables such as interest rates, economic indicators, or regional demand factors to improve prediction accuracy — especially for high-variance markets like California.
